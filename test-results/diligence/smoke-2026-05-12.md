# Diligence Smoke Test — 2026-05-12

Module-level smoke run against the `main` branch (commit `09975a7`) after the
mood/energy rename and the de-AI pass on comments and docs. Purpose: confirm
the renamed code paths work end-to-end without needing an LLM or environment
server hooked up.

## What was exercised

Five core modules, instantiated directly with mocked dependencies:

- `src/cognition/InternalState.js` — mood/energy update, describe(), salience,
  creativity feedback, history summary
- `src/memory/WorkingMemory.js` — push with salience, action+result merge,
  recent() formatting
- `src/cognition/RepetitionGuard.js` — record, check() warnings, isFixated,
  scoreSpeech
- `src/cognition/DeltaDetector.js` — detect across two consecutive ticks,
  narrate the deltas
- `src/cognition/Perceive.js` — full observation → natural language narration

The raw output is preserved in `smoke-2026-05-12.txt` next to this file. The
script that produced it is at the end of this document.

### InternalState

after positive tick:     mood=0.125 (neutral) energy=0.209 (elevated)
description:             A comfortable focus — present and attentive
salience:                0.605
after failure:           mood=-0.038 energy=0.188
after repetitive speech: mood=-0.118
after novel speech:      mood=-0.088
history summary:         Average mood: 0.04, average energy: 0.20.
                         Peak energy: 0.21. Mood range: -0.04 to 0.12.

**Observed behaviour matches the documented design:**

- Asymmetric reward visible: a single failure dropped mood from +0.125 to
  -0.038 (a delta of -0.163, consistent with the -0.15 failure nudge plus
  ~10% decay).
- Speech creativity feedback applies through the affect channel: a
  repetitive utterance (score < 0.4) shifted mood by -0.08; a novel one
  (score > 0.8) nudged it back by +0.03. 2.7:1 ratio matches v0.3.6 spec.
- `historySummary()` produces the post-rename string format (`Average mood`,
  `average energy`, `Peak energy`, `Mood range`) — no straggling references
  to the old field names.

Three events pushed (`action`, `action_result`, `speech_sent`) — buffer

### RepetitionGuard

```
after 4x move_to(wander): 3 warnings
  - You have done "move_to" three times in a row. Try something different.
  - You targeted "wander" 4 out of your last 5 actions. Try something different.
  - You keep doing exactly the same thing with the same parameters. Break the pattern.
isFixated(move_to, wander): false  (history < 10 entries, threshold check is conservative)
scoreSpeech("hello there"): 1.00
scoreSpeech("hello there") after recording: 0.50
```

Three independent detection patterns fired correctly on 4 consecutive
identical actions: consecutive repeat, target dominance, exact-key
duplicate.

The post-record `scoreSpeech` of 0.50 (rather than 0.0) is correct
behaviour: after stop-word removal, "hello there" reduces to a single
keyword (`hello`), and the scoring function returns 0.5 for messages with
fewer than 2 meaningful keywords. Exact-match detection only fires when
the message has enough lexical content to compare against.

## Overall

All five modules produce expected output post-rename and post-de-AI. No
references to the old `valence`/`arousal` field names remain in the
runtime. `node --check` passes clean on all .js files at the repo root
and under `src/`.

The full automated test suite (`test-suite.js`) requires a running agent
connected to a WebSocket env server and takes ~12 minutes to complete its
10 scenarios. It hasn't been run against the renamed code in this session
because of the LLM dependency. Recommended next step before delivery: run
`test-suite.js` once on a Pi (or any host with a Groq key) and add the
report to `test-results/diligence/`.

## Reproduction

```bash
cd agent-runtime
npm install
node -e "
import('./src/cognition/InternalState.js').then(async (m) => {
  const logger = { info:()=>{}, warn:()=>{}, error:console.error, debug:()=>{} }
  const s = new m.InternalState(
    { stateDecayRate: 0.1, signalPullRate: 0.15, dataDir: '/tmp/3aiii-smoke' },
    logger
  )
  s.update({
    actionResult: { success: true, action: 'interact' },
    deltas: [{ type: 'appeared', category: 'object', id: 'thing-01' }],
    environmentSignals: { vitality: 0.7 },
    worldEvents: [],
  })
  console.log(JSON.stringify(s.describe(), null, 2))
})"
```

Expected output: a JSON object with `mood`, `energy`, `moodLabel`,
`energyLabel`, and `description` fields. Values will differ on each run
depending on timing, but the shape and field names are stable.
