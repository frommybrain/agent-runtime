# Agent Runtime — Status & Plan (2026-02-28)

## Where We Are

### agent-runtime (this repo)
**Status: Feature-complete, ready for Pi deployment**

Core architecture fully implemented:
- OBSERVE → THINK → ACT loop on 8s heartbeat
- 3-tier memory: persistent markdown (memory.md, skills.md, tools.md) + working RAM ring buffer + daily logs (7-day GC)
- Dual LLM: Ollama primary (local) → cloud API fallback → heuristic fallback brain
- Sleep cycle: 4hr active → 1hr LLM-driven memory consolidation
- HTTP API with SSE streaming (/status, /memory, /events, /logs/today, /sleep, /wake, /persona)
- Pi bootstrap script (setup-pi.sh) — Node 20, Ollama, systemd service, auto-update cron
- 4 personas: Pip, Bean, Mochi, Taro (renaming to new names — first agent is "Victor")
- Default model: qwen2.5:3b (better JSON reliability than llama3.2:3b)

WebSocket protocol (agent ↔ environment):
```
IDENTIFY → IDENTIFIED
OBSERVE  → OBSERVATION (with available_actions)
ACT      → ACTION_RESULT
WORLD_EVENT (async: speech, joins, terminal output)
```

### anon-ai-world (3D world)
**Status: Working sim with embedded LLM cognition, not yet integrated with agent-runtime**

- sim-server (port 3999): authoritative world engine, 20Hz physics, 10Hz broadcast, 4 NPC kiwi birds, needs/drives/mood, LLM via Groq/Ollama, persistent memory, SQLite event logging
- viewer (Next.js port 3000): React Three Fiber WebGPU renderer, HUD, event feed, minimap

The sim-server has its OWN embedded cognition pipeline. agent-runtime was extracted from it to be portable. They are NOT yet integrated — sim-server doesn't speak the agent-runtime WebSocket protocol.

## The Gap

To plug agent-runtime (on Pi) into anon-ai-world:
1. sim-server drives NPC brains internally (LLMBrain.js → Pipeline.js)
2. agent-runtime expects IDENTIFY/OBSERVE/ACT WebSocket protocol
3. sim-server doesn't expose that protocol — only broadcasts viewer snapshots/deltas

**Need**: An agent API endpoint on the sim-server (or adapter) that translates between world state and the agent-runtime protocol.
